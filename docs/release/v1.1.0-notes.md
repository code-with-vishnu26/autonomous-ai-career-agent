# v1.1.0 ‚Äî Portability, Packaging & Onboarding Hardening

**Software release state:** `RELEASED` ‚Äî the annotated `v1.1.0` tag was
manually created and pushed by the maintainer, and the GitHub Release is
published (see [Post-release reconciliation](#post-release-reconciliation-phase-45)
below). **Product safety posture:** `PREPARE_ONLY` (unchanged) ‚Äî the tool
still prepares up to a human confirmation and never submits to any external

**Promoted from:** [`v1.0.0`](v1.0.0-notes.md) (Phases 43‚Äì44 / ADR-0062;

This is a minor release on top of the stable v1.0.0. It keeps the
supervised, prepare-only discover ‚Üí decide ‚Üí ingest ‚Üí prepare ‚Üí learn
workflow exactly as it was ‚Äî **it still does not submit applications to any
external system** ‚Äî and adds one backward-compatible configuration surface
plus a set of packaging, onboarding, and documentation-accuracy fixes.

## Why this is a MINOR bump (SemVer)

The only runtime source change since v1.0.0 is a new, optional,
`.env`-overridable `Settings` field. Per SemVer 2.0.0 that is new
backward-compatible functionality ‚Üí MINOR. See
[ADR-0062](../adr/0062-v1-1-readiness-audit-and-version-decision.md) for the
full audit and reasoning (why not v1.0.1, why not "remain unreleased").

## Behavior compatibility

For the **documented, supported** usage (editable install, run from the repo
root), `promptfoo/results` resolves to exactly the same path as before ‚Äî
**no behavior change.** The change is a strict bugfix for the previously
broken wheel/non-editable case, plus one new optional config knob. **No
dependency versions changed.**

## Live-LLM evidence

**No fresh live-LLM run was required for this release, and none is claimed.**
No LLM-facing code changed since v1.0.0 ‚Äî the truthfulness gate, both
`ClaimVerifier` implementations, the prompt text, and the promptfoo prompt
version (`truthfulness-gate-v2`) are byte-identical to v1.0.0. The v1.0.0
controlled live-Groq evidence (Phase 36 / ADR-0059) therefore carries
forward unchanged. As with v1.0.0, CI itself makes **no** live LLM call (no
key, egress blocked by design).

## Capability matrix and known limitations

Unchanged from v1.0.0. Still true:

1. PDF/OCR CV import: **NOT_SUPPORTED**.
2. Browser/email/autonomous external submission: **NOT_SUPPORTED** ‚Äî code
   exists, unwired, unreachable (`executor_available=False`, hardcoded).
3. macOS: **untested** ‚Äî a deliberate, documented gap.
4. PDF rendering may require a local LibreOffice install.

## Upgrade / install

```bash
git pull
pip install -e ".[dev]"
career-agent setup
```

No database migration is required. No configuration change is required: the
new `promptfoo_results_dir` defaults to the same `promptfoo/results` path
the tool already used for supported usage.

## Release decision

**GO** for v1.1.0 ‚Äî and now **released** (see below). Every release-blocking
invariant that held for v1.0.0 still holds (architecture contracts, the
prepare-only execution boundary, the fabrication and Promptfoo gates, no
secret/private-data leak, external submission unreachable); the full test
suite and CI are green on both supported platforms for the commit this
version is built from; and no LLM-facing behavior changed. See ADR-0062 for
the readiness audit, the SemVer reasoning, and the risk register (no P0/P1).
