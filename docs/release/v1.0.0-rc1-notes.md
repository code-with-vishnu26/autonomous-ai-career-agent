# v1.0.0-rc1 — Supervised Prepare-Only Release Candidate

**Status:** Release candidate. **Position:** `PREPARE_ONLY`. **Date:** 2026-07-10.

This is the first release candidate for the Autonomous AI Career Agent. It packages
the discover → decide → ingest → prepare → learn workflow as a supervised,
prepare-only tool. **It does not submit applications to any external system.**

## Known limitations

1. **No external submission.** By design for v1.0. Submission code exists but is
   unreachable from the CLI; every run ends at a confirmation boundary.
2. **Live LLM output quality is not validated in CI.** CI/tests make no real LLM
   calls. Real-output quality requires the user's local controlled smoke run
   (ADR-0055).
3. **Live Promptfoo validation is machine-local.** No `promptfoo/results/`
   artifact is committed; verifier validation against a real provider is
   BLOCKED_BY_CONFIGURATION in any environment without a key + artifact.
4. **No CI in this repository.** Lint/architecture/tests/packaging must be run
   manually per `RELEASE_CHECKLIST.md`.
5. **Windows/macOS not executed here.** Windows UTF-8 behavior is enforced
   statically via explicit `encoding="utf-8"`; actual Windows and macOS runs are
   untested.
6. **PDF rendering may require LibreOffice.** DOCX artifacts always render; PDF
   conversion depends on a local LibreOffice being present.
7. **PDF/OCR CV import unsupported.** Use `.docx`, `.txt`, or `.md`.

## Post-rc1 updates

- **Phase 36 (ADR-0058):** the first real, live-Groq controlled smoke (run
  by the maintainer on a real Windows machine with a real `GROQ_API_KEY`)
  found and fixed a real idempotency-lifecycle defect: declining the
  confirmation prompt left the application-store row stuck at
  `status="pending"`, permanently refusing every future retry for that
  opportunity. Fixed by recording each run's terminal status correctly
  (`"declined"` is now excluded from the retry-block set, the same as
  `"rejected"` — both mean zero external side effect). Genuinely risky
  statuses remain fully blocking. No schema migration; no truthfulness,
  ATS, Promptfoo, journal, or submission-reachability semantics changed.

## Upgrade / install

```bash
pip install -e ".[dev]"
career-agent setup
```

No database migration is required for any change through Phase 36 (the
`applications.status` column has no `CHECK` constraint, so the new
`"declined"` value is compatible with every existing database). Aside from
the idempotency-lifecycle fix noted above, nothing in this release changes
truthfulness, Promptfoo, ATS, journal, or submission-reachability
semantics.
